# flight-booking

A REST API for flight booking built with Spring Boot 3.4. Provides endpoints to list available flights, create bookings (with thread-safe seat allocation), and retrieve booking details. Uses an OpenAPI-first approach — DTOs are generated from `api-spec.yaml` via the OpenAPI Generator Maven plugin.

## Tech Stack

- Java 21, Spring Boot 3.4.0, Maven
- SpringDoc OpenAPI (`springdoc-openapi-starter-webmvc-ui 2.8.9`)
- OpenAPI Generator Maven Plugin 7.11.0
- JUnit 5 + AssertJ (tests)
- In-memory storage (`ConcurrentHashMap`)

## Project Structure

```
src/main/java/com/flightbooking/
  config/          DataPreloader — seeds 4 flights on startup
  controller/      BookingController — REST endpoints under /api/v1
  dto/gen/         Generated DTOs (BookingRequest, BookingResponse, FlightResponse)
  exception/       GlobalExceptionHandler + custom exceptions (FlightNotFound, BookingNotFound, Overbooking)
  model/           Domain classes (Flight, Booking, BookingStatus)
  repository/      In-memory FlightRepository and BookingRepository
  service/         BookingService — business logic with synchronized seat allocation
```

## How to Run

```bash
# Generate DTOs from the OpenAPI spec and compile everything
mvn compile

# Start the server on port 8080 (4 flights pre-loaded)
mvn spring-boot:run

# Run all 22 tests
mvn test
```

Swagger UI is available at [http://localhost:8080/swagger-ui.html](http://localhost:8080/swagger-ui.html)

## Example Requests

### Create a booking

```bash
curl -X POST http://localhost:8080/api/v1/bookings \
  -H "Content-Type: application/json" \
  -d "{\"flightId\":1,\"passengerName\":\"Jane Doe\",\"passengerEmail\":\"jane@example.com\",\"seatCount\":2}"
```

**Response** — `201 Created`
```json
{
  "id": "a1b2c3d4-...",
  "flightId": 1,
  "flightNumber": "BA123",
  "passengerName": "Jane Doe",
  "passengerEmail": "jane@example.com",
  "seatCount": 2,
  "bookingTime": "2026-07-08T12:00:00Z",
  "status": "CONFIRMED"
}
```

### Validation error example

```bash
curl -X POST http://localhost:8080/api/v1/bookings \
  -H "Content-Type: application/json" \
  -d "{\"flightId\":0,\"passengerName\":\"\",\"passengerEmail\":\"bad\",\"seatCount\":0}"
```

**Response** — `422 Unprocessable Entity`
```json
{
  "type": "error/validation",
  "title": "Validation Error",
  "status": 422,
  "detail": "Validation failed",
  "instance": "/api/v1/bookings",
  "timestamp": "2026-07-08T12:00:00Z",
  "path": "/api/v1/bookings",
  "errors": [
    {"field": "passengerName", "message": "size must be between 2 and 100"},
    {"field": "passengerEmail", "message": "must be a well-formed email address"},
    {"field": "seatCount", "message": "must be greater than or equal to 1"},
    {"field": "flightId", "message": "must be greater than or equal to 1"}
  ]
}
```

### Overbooking example

```bash
curl -X POST http://localhost:8080/api/v1/bookings \
  -H "Content-Type: application/json" \
  -d "{\"flightId\":1,\"passengerName\":\"John\",\"passengerEmail\":\"john@test.com\",\"seatCount\":999}"
```

**Response** — `409 Conflict`
```json
{
  "type": "error/overbooking",
  "title": "Overbooking",
  "status": 409,
  "detail": "Not enough seats available on flight BA123. Requested: 999, Available: 200",
  "instance": "/api/v1/bookings",
  "timestamp": "2026-07-08T12:00:00Z",
  "path": "/api/v1/bookings"
}
```

## Key Design Decisions

- **In-memory storage**: `ConcurrentHashMap`-backed repositories — no database setup required, data resets on restart
- **Thread-safe bookings**: `synchronized (flightLock)` in `createBooking()` ensures atomic seat look-up, capacity check, and seat-count update under concurrent access
- **OpenAPI-first models**: DTOs are generated by `openapi-generator-maven-plugin` from `src/main/resources/api-spec.yaml`; hand-written DTOs were removed
- **Error responses**: RFC 9457 `ProblemDetail` with `type`, `title`, `status`, `detail`, `instance`, `timestamp`, `path`, and a structured `errors` array for validation failures
- **HTTP status codes**: 201 (created), 400 (malformed request), 404 (not found), 405 (wrong method), 409 (overbooking), 422 (validation failure), 500 (unhandled error)
- **No Mockito in tests**: Uses real repository instances directly to avoid inline-mock-maker incompatibility with Java 25

## What I'd Improve Given More Time

- **Persistent storage**: Replace in-memory repositories with PostgreSQL + JPA/Hibernate so data survives restarts
- **Pagination / sorting / filtering**: Add pageable list endpoints for flights and bookings
- **More endpoints**: Booking cancellation, modification, and flight search by origin/destination/date
- **Authentication & authorization**: JWT-based security with user roles (admin vs. passenger)
- **Integration tests**: Test the full HTTP lifecycle with `@SpringBootTest(webEnvironment = RANDOM_PORT)` and `TestRestTemplate`
- **Contract tests**: Use Pact or Spring Cloud Contract to validate API compatibility with consumers
- **Docker**: Dockerfile and `docker-compose.yml` for local development
- **CI/CD**: GitHub Actions pipeline for build, test, and deploy
- **Observability**: Micrometer metrics, structured JSON logging, distributed tracing with OpenTelemetry
- **Rate limiting & idempotency**: Protect against accidental duplicate bookings and abuse
- **Multi-module Maven project**: Split into `api`, `domain`, and `infrastructure` modules for clearer separation of concerns
